How can I ensure that my code, program, or software is functional and secure?

I ensure my software is functional and secure by writing thorough unit tests and verifying that each requirement is met before considering a project complete. Throughout this course, I used JUnit to test both normal functionality and edge cases, helping me catch errors early in development. I also focused on input validation and preventing invalid data from being accepted, which improves both reliability and security. Testing consistently throughout development makes it much easier to identify and fix problems before software is released.

How do I interpret user needs and incorporate them into a program?

I begin by carefully reviewing the software requirements and breaking them into smaller, manageable tasks. Each requirement is translated into specific classes, methods, and tests to ensure the finished program behaves exactly as expected. Writing unit tests based directly on the requirements helped me verify that every feature worked correctly and that future changes would not unintentionally break existing functionality.

How do I approach designing software?

I approach software design by planning the structure before writing code. I divide the program into smaller, focused classes that each have a single responsibility, making the code easier to understand, maintain, and test. During this course, I designed separate service and model classes for contacts, tasks, and appointments, which kept the code organized and allowed each component to be tested independently. I also consider maintainability and future updates so the software can be expanded without requiring major changes to the existing design.
